Spacing ​

Cobalt uses a 4 px base unit spacing scale. Every spacing token is a multiple of 4, which keeps layouts aligned to a consistent grid and simplifies visual rhythm across components and pages.

Scale ​

TokenValueCommon use
--co-spacing-44 pxInline icon gap, tight padding
--co-spacing-88 pxInput padding (inline), badge padding
--co-spacing-1212 pxCompact component padding
--co-spacing-1616 pxDefault component padding, card body
--co-spacing-2020 pxForm field gap
--co-spacing-2424 pxSection padding on small screens
--co-spacing-3232 pxCard padding, modal body
--co-spacing-4040 pxSection padding on medium screens
--co-spacing-4848 pxLarge section gaps
--co-spacing-6464 pxPage-level section separation
--co-spacing-8080 pxHero area padding
--co-spacing-9696 pxMaximum section padding on wide viewports

Usage in CSS ​

Reference spacing tokens with var():

css
.co-card {
  padding: var(--co-spacing-24);
  gap: var(--co-spacing-16);
  display: flex;
  flex-direction: column;
}

.co-stack > * + * {
  margin-top: var(--co-spacing-16);
}

Tokens also work in calc() expressions when you need derived values:

css
.co-sidebar {
  width: calc(var(--co-spacing-64) * 4); /* 256 px */
  padding: var(--co-spacing-24) var(--co-spacing-16);
}

Choosing the right size ​

Use these guidelines to decide which token fits your context:

ContextRecommended tokens
Inline elements (icon + label)--co-spacing-4 to --co-spacing-8
Inside compact controls (badges, chips)--co-spacing-4 to --co-spacing-8
Inside standard controls (buttons, inputs)--co-spacing-8 to --co-spacing-12
Component internal padding (cards, alerts)--co-spacing-16 to --co-spacing-24
Between sibling components--co-spacing-16 to --co-spacing-32
Page sections--co-spacing-48 to --co-spacing-96

Tip: When in doubt, start with --co-spacing-16 and adjust up or down by one step. Jumping more than two steps between adjacent elements usually signals a missing layout container.

Responsive spacing ​

Spacing tokens are static values, but you can combine them with breakpoint media queries to scale padding at different viewport sizes:

css
.co-section {
  padding-block: var(--co-spacing-32);
}

@media (min-width: 768px) {
  .co-section {
    padding-block: var(--co-spacing-48);
  }
}

@media (min-width: 1280px) {
  .co-section {
    padding-block: var(--co-spacing-80);
  }
}

Best practices ​

  1. Stick to the scale. Avoid arbitrary values like 13px or 1.125rem — they break the 4 px grid.
  2. Use logical properties. Prefer padding-inline and margin-block over padding-left for better RTL support.
  3. Keep related elements closer. The Gestalt principle of proximity applies — tighter spacing signals grouping.
  4. Be generous with whitespace at the page level. Sections separated by --co-spacing-64 or more feel less cluttered and improve scanning.
