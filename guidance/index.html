Guidance ​

Cobalt Design System provides more than just components — it offers a set of guidelines, principles, and best practices to help your team build cohesive, high-quality experiences.

Overview ​

These resources are intended for designers, developers, and content authors working with Cobalt. Whether you are starting a brand-new project or migrating an existing application, the guidance section will help you make informed decisions at every stage.

Sections ​

PageDescription
Design PrinciplesThe core values that drive every decision in Cobalt.
Content & WritingVoice, tone, capitalization, error messages, and formatting rules.
Developer OnboardingEnvironment setup, project structure, and first steps with Cobalt.
Migration GuidesStep-by-step instructions for moving from Bootstrap, Material UI, or raw HTML/CSS.

Who is this for? ​

  • Designers — Understand the reasoning behind component behavior and layout decisions.
  • Developers — Get up and running quickly, learn conventions, and avoid common pitfalls.
  • Content authors — Write clear, consistent copy that aligns with the Cobalt voice.
  • Tech leads — Plan migrations and evaluate Cobalt for your team's needs.

How to use these guides ​

Each page is self-contained. Start with Design Principles if you want to understand the philosophy behind the system, or jump straight to Developer Onboarding if you are ready to write code.

Tip: Bookmark the Content & Writing page — it is the most frequently referenced guide during design reviews.

Contributing ​

If you spot an error or want to propose a new guideline, open an issue in the Cobalt repository with the guidance label. All changes to this section go through the same review process as component code.
